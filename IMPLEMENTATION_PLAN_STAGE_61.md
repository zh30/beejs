## 🧪 测试驱动开发流程

### Red-Green-Refactor 循环

#### 1. Red (写失败的测试)
```rust
#[test]
fn test_v8_initialization() {
    let runtime = RuntimeLite::new();
    assert!(runtime.is_v8_initialized());
}
```

#### 2. Green (写最少的代码让测试通过)
```rust
pub fn is_v8_initialized(&self) -> bool {
    self.v8_initialized
}
```

#### 3. Refactor (重构代码)
- 提取公共代码
- 改善命名
